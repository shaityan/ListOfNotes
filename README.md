## Название проекта

**Список заметок**

Простое мобильное приложение для создания и просмотра текстовых заметок. Приложение рассчитано на одного пользователя и не требует регистрации. Основной упор — на базовый функционал, понятную навигацию и простую реализацию в рамках лабораторных работ.

---

## Основная идея и цели

- **Цель**: дать пользователю быстрый способ записывать короткие заметки (дела, идеи, напоминания) и просматривать их в удобном списке.
- **Фокус**: минимализм, простая структура экранов, базовый ввод текста, хранение заметок локально и один пример запроса к публичному API.

---

## Основные функции приложения

1. **Просмотр списка заметок**
   - Отображение всех сохранённых заметок в виде списка.
   - Каждая заметка показывает заголовок и первые строки текста / дату.

2. **Создание новой заметки**
   - Экран с полями: заголовок (необязательно) и текст заметки (обязательно).
   - Кнопка "Сохранить" добавляет заметку в список.

3. **Просмотр и редактирование заметки**
   - При нажатии на заметку из списка открывается экран с полным текстом.
   - Возможность отредактировать текст и сохранить изменения.

4. **Удаление заметки**
   - Кнопка "Удалить" на экране заметки (с простым подтверждением).

5. **Хранение данных**
   - ЛР6 — сохранение заметок с помощью `shared_preferences`.

6. **Работа с публичным API**
   - Кнопка "Случайная идея" на экране создания заметки.
   - По нажатию выполняется HTTP-запрос к открытому API цитат/фактов и подставляется текст в поле заметки или в отдельный блок "Подсказка".

7. **Стилизация**
   - Простая светлая тема: светлый фон, акцентный цвет кнопок.
   - Единый стиль заголовков, списков и кнопок.

---

## Структура экранов

Приложение содержит минимум два основных экрана:

1. **Главный экран — список заметок**
   - Заголовок: "Мои заметки".
   - Кнопка "+" (добавить заметку).
   - Список карточек заметок:
     - Заголовок (если есть).
     - Пара первых строк текста или дата создания.

2. **Экран создания / редактирования заметки**
   - Заголовок: "Новая заметка" или "Редактирование заметки".
   - Поле ввода заголовка (однострочное, необязательное).
   - Поле ввода текста (многострочное).
   - Кнопка "Сохранить".
   - Кнопка "Случайная идея" (запрос к API и подстановка текста).
   - (Для уже существующей заметки) кнопка "Удалить".

## Навигация между экранами

- **Из главного экрана в экран создания заметки**:
  - Нажатие на кнопку "+".
- **Из главного экрана в экран просмотра/редактирования заметки**:
  - Нажатие на элемент списка (карточку заметки).
- **Возврат назад**:
  - Кнопка "Назад" в верхней части экрана или системная кнопка.
- **Удаление заметки**:
  - На экране заметки — кнопка "Удалить" — подтверждение — возврат на главный экран.

Схема навигации (упрощённо):

```text
[Список заметок] --(клик по +)--> [Новая заметка]
      |                                   |
      | (клик по заметке)                 |
      v                                   v
 [Просмотр/редактирование заметки] --(Назад)--> [Список заметок]
```

## Сценарии использования

1. **Создание новой заметки**
   - Пользователь открывает приложение → видит список (в начале может быть пустым).
   - Нажимает на кнопку "+".
   - Вводит заголовок и текст заметки.
   - Нажимает "Сохранить".
   - Новая заметка появляется в списке.

2. **Использование публичного API (подсказка)**
   - На экране новой заметки пользователь нажимает "Случайная идея".
   - Приложение отправляет HTTP-запрос к открытому API (например, цитата дня).
   - Полученный текст отображается:
     - либо вставляется в поле текста заметки,
     - либо показывается ниже как "Предложение для заметки".
   - Пользователь при необходимости редактирует текст и нажимает "Сохранить".

3. **Просмотр и редактирование заметки**
   - На главном экране пользователь нажимает на существующую заметку.
   - Открывается экран с полным текстом.
   - Пользователь меняет текст и/или заголовок.
   - Нажимает "Сохранить" — изменения отображаются в списке.

4. **Удаление заметки**
   - Пользователь открывает заметку.
   - Нажимает "Удалить".
   - Подтверждает удаление (простое диалоговое окно).
   - Приложение возвращается на список, заметка исчезает.

## Разделение по лабораторным работам

### ЛР4 — Верстка

- **Цели ЛР4**:
  - Создать базовую структуру экранов без полноценной логики.
  - Разметить:
    - главный экран со списком (пока можно использовать заглушки — статический список),
    - экран создания/редактирования заметки (поля ввода, кнопки).
  - Настроить навигацию между экранами (переходы по кнопкам).
  - Добавить простую цветовую схему и базовое оформление.

**Результат ЛР4**:
- Приложение переключается между 2–3 экранами.
- На экранах есть визуально оформленные элементы (список, поля ввода, кнопки).

### ЛР5 — Логика

- **Цели ЛР5**:
  - Реализовать добавление заметок:
    - нажатие "Сохранить" создаёт объект заметки и добавляет его в список в памяти.
  - Реализовать редактирование и удаление заметок:
    - корректное обновление списка на главном экране.
  - Реализовать обработку нажатий на список (открытие заметки).
  - Подключить HTTP-клиент и реализовать запрос к публичному API для кнопки "Случайная идея".

**Результат ЛР5**:
- Приложение работает "наживую": можно добавлять, изменять и удалять заметки в рамках текущего запуска.
- Работает кнопка запроса к API и подстановка полученных данных в интерфейс.

### ЛР6 — Данные

- **Цели ЛР6**:
  - Добавить постоянное хранение заметок:
    - с помощью `shared_preferences` (список заметок в виде JSON).
  - При запуске приложения:
    - загрузка заметок из хранилища,
    - отображение их в списке.
  - При добавлении/редактировании/удалении:
    - обновление данных в хранилище.

**Результат ЛР6**:
- Все заметки сохраняются между перезапусками приложения.
- Основной функционал списка заметок полностью готов (MVP).

## Мини-чек-лист требований

- **Экраны**:
  - Есть главный экран со списком заметок.
  - Есть дополнительный экран (создание/редактирование заметки).
- **Навигация**:
  - Переход со списка к созданию заметки.
  - Переход со списка к редактированию существующей заметки.
  - Возврат назад работает.
- **Интерактивность**:
  - Есть минимум одно текстовое поле (заголовок/текст).
  - Есть кнопки "Сохранить", "Случайная идея", "Удалить".
- **Список данных**:
  - Список заметок отображается на главном экране.
- **Хранение данных**:
  - На этапе ЛР5 — в памяти.
  - На этапе ЛР6 — в `shared_preferences` или БД.
- **Работа с API**:
  - Есть хотя бы один запрос к открытомy HTTP API.
- **Стилизация**:
  - Определены основные цвета, отступы и стиль кнопок/заголовков.
